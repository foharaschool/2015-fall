Lab Specifications

Lab Overview

In this lab you will be writing a JavaScript program that, each time the submit button on the form is clicked, will accept a string from a text box on a form, and save the string into an array. The array will increase by one string each time the submit button is clicked, thus gradually increasing the number of 'slots' in the array (each 'slot' holding a string). In addition to acquiring another string, also each time the submit button clicked, the entire contents of the array as it exists at that point in time must be displayed on the Web page.  Note: There is just one function involved with this lab, namely the main function named:  lab07firstArrays().

Lab 07 Steps

  1. In the HTML file, create a form with one input text box field and a submit button.

    [Note: While definitely not in general a good way to proceed, if the CSS style sheet for a Web page is unavailable, an embedded style sheet could be set up in the Head element of the Web page itself as kind of "quick and dirty" way to place CSS rules into effect. We could do that here to make the form more presentable, but it is not required. And to repeat: in a well-organized Web site, all CSS rules belong in the usual external style sheet file(s), and should not be set up within HTML files themselves.]
  2. Give the form an   id   attribute, or, give the text box field itself an  id  attribute (which one depends upon how you later wish to generate a reference to the string keyed into the text box).
  3. Within the output <div> tag set already present in the HTML source code, add an HTML <ol> element (aka an <ol> tag set) to allow line items to be inserted later by the program, and thus displayed on the web page.
  4. Give the <ol> element an id attribute, with a value of something like "theOrderedList".  

    [Note: The purpose of this id attribute is to later be able to reference the <ol> element via use of the getElementById() method.]
  5. Give an onsubmit attribute (aka event handler) to the form, an attribute which calls the lab07firstArrays() function.. Remember from course 10-152-120 that this attribute goes on the opening <form> tag, and not the submit button (i.e. the <input> element that creates the submit button).
  6. Now working with both the source code in the HTML file and the source code in the JavaScript file /unit5/jsFiles/lab07firstArrays.js. test to see if a click of the submit button does in "fire" the function "lab07firstArrays()". And furthermore, note that, as currently coded, the function is supposed to be displaying a message on the web page.  If it does not do this (and the chances are that it does in fact not do this), debug this situation right now, before beginning to change the function to accomplish Step #8 below.

    [Note: What is needed here is similar to what was required in Lab 06: namely, having the function return a value of false , and then having that value passed on into the programming code underlying the onsubmit event.]
  7. Create a global array variable to hold the strings that will be entered by the viewer. Note that it must be global so that it will preserve the array elements that have already been added by previous clicks of the submit button. If it was local within the function, it would be wiped back to empty every time the viewer ran the function. Initialize it when you declare it as follows, using one of the two suggested names below:
    • var enteredStrings = [];
      •        or
    • var enteredStringsArray = [];

    [Note:  Arrays can be named using any characters. The word "Array" need not be a part of the name of arrays, and its presence or absence has no meaning to any software. However, including it might be a useful convention for programmer comprehension when later writing code involving arrays. If you decide to include it as part of the name, just make sure that you don't come to believe that it must be used when naming arrays, or become psychologically dependent on its presence.]
  8. When the form is submitted, get the value in the text field and insert it at the end of the array.

    [Note: When accomplishing this Step #8, make liberal use of the alert() method to verify to what any given variable or expression is referring. Also, note that this placement at the end of the array can be accomplished either by working with the length property of the array, or, by employing the push() function.]
  9. In the lab07firstArrays() function, create a local variable to hold a reference to the HTML display element (the <ol>) set up in Step #3 above.
  10. Get a reference to the HTML display element (the <ol>) and assign it to the local variable created in Step #9 just above.
  11. Create a loop that goes through the array and outputs the content of each array element as a line item (an <li>) into the HTML display element. This will require use of the innerHTML property.

    [Note: Two approaches to accomplishing this suggest themselves. One would be to place via the plus asign operator (+=) all the line items into a variable, and then when the looping has finished, assign that variable's content into the <ol>. Since the asign operator (=) is an "erase/replace" operation, any previous content within the <ol> would be automatically wiped away. The other approach would be to clear away any previous display on the web page within the <ol> before running this loop. And then using the plus asign operator (+=), place the line items into the <ol> one at a time within the loop itself.    ]
  12. Run a final test of your code by inputting several strings, and see if they are showing up on the web page.

Note:  Keep in mind that when a Web page is refreshed (by a press of F5), not only is a fresh copy of the HTML source code for the Web page employed to re-render the Web page, but also, a fresh copy of any external JavaScript file source code referenced by that Web page is picked up at that point.  Thus, so long as the Web page for Lab 07 is NOT refreshed, the array building up within the JavaScript library file associated with Lab 07 will remain in existence.   It is important to realize that clicking the submit button on a rendered Web page only runs the JavaScript program associated with the onsubmit event handler--it does NOT refresh the Web page (like a press of F5 would do).
