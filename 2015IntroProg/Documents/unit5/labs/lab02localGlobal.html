Lab Steps

  1. Open this file in Firefox and jEdit.
  2. Add your name, section, and email.
  3. Open the file /unit5/jsFiles/lab02localGlobal.js in jEdit.
  4. Open the HTML file /unit5/labs/lab02localGlobal.html and find the <ol id="outputList"> tag. You will be adding <li> tags to this with JavaScript.
  5. Now switch to the JavaScript library file. Add a global variable named fullName at the top of the file.
  6. Below the global variable, create the 'skeleton' of a new function named  printFullName(). Don't forget the closing curly brace.  [NOTE: Make sure that the flow of control is able to enter this new function by at this time placing a call to the  printFullName() function into the main function. Only when you have verified that the flow of control is successfully entering the new function should you go on to the next step in this lab.]
  7. In the new  printFullName() function:
    1. Create a local variable also named fullName.
    2. Assign the local variable the value, "Bill Smith".
    3. Add a list item to the HTML ordered list with the id of "outputList" with the local fullName variable as the data.
  8. In the main function lab02localGlobal():
    1. Assign the name "Judy Green" to the global variable version of fullName.
    2. Add a list item to the HTML ordered list with the id of "outputList" with the global fullName variable as the data.
    3. Call the  printFullName() function.
  9. Inside of a function, if there is a local variable there that has the exact, same name as a global variable, which variable (i.e. the global one or the local one) has precedence? That is, if some JavaScript program code makes a reference to the name of that variable, will the program use the local one, or, will it use the global one?
