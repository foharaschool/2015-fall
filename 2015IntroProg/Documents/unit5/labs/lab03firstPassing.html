  • Lab Steps
    1. Create the "skeleton" of a new function above the main function lab03firstPassing();. Don't forget to set up a comment just above it, with a one sentence first draft telling the purpose of this new function.
    2. Name the new function displayCity().
    3. For version one of the new function, place in it nothing but an alert() with text that says something like "Flow of control entered the new function".
    4. Then in the main function lab03firstPassing(), place the call to the function displayCity(). Don't forget to include the parentheses--they are a part of the name of the function, and must be there, even though there is nothing within them for the moment.
    5. Test the program [remember that a program is a "set of functions"], to make sure the flow of control is successully entering the new function. Test this to make sure it is working before going ahead and attempting to do anything else with this new function.
    6. Now, in the function lab03firstPassing(), prompt the user for a city name. Use a local variable to hold the city keyed in by the user. Test it.
    7. In the new displayCity() function, add a parameter within the parentheses of the new function--name iit something like "cityReceived". Note that this creates a function variable, which is essentially a local variable available only within the new displayCity() function.
    8. Then back in the main function lab03firstPassing(), modify the call to the displayCity() function as follows. Place the name of the local variable holding the city name inside the parentheses associated with the name of the new function. A variable name positioned in this fashion is often referred to as an argument.
    9. Before doing anything further, make sure that the value of the variable holding the city name keyed in by the user has been successfully passed into the new function. Do this by modifying the alert() so that instead of showing the text about the flow of control entering the new function , the alert()now shows the value of the "cityReceived" function variable. Test this to make sure it is working before going ahead and attempting to do anything else with this received value.
    10. Now, in the new function, output a city greeting to the output div tag in the HTML file. The greeting must contain the city name, with some text added before and after the city name. For example, assuming the user keyed in "Cheddarville" the greeting might be something like, "Hello, Cheddarville. How's the weather out there?" Feel free to use any greeting you want as long as there is text before and after the city name.
    11. Then again, run the final test of the application.
    12. The last action to take is to now modfy the comment box above the new function. Remember that the original content there was only first draft. Now, with very fresh knowledge of what the function does, revise this comment for completeness, accuracy, and good sentence structure.

  • Optional if time:  Add in a 'guard dog' IF at the top of the new function to make sure that it was called with the correct number of parameters, and if not, then have it throw an error. This would involve interrogating the arguments.length object and property, and then using a throw statement.
