It's Moving Day!

We're moving on. We liked it in our old place but it got to be too cramped. We want space.

  • We're going to move our JavaScript code out of our HTML files. This is done with functions.
  • Functions are just a bunch of JavaScript code that is collected together in a location and given a name.
  • Functions are normally named using one or more words (in camel case), with the first word being a verb. But there is nothing in the software that enforces this--this is merely a convention among programmers. And in fact, for this course, the main function being invoked from between the script tags in the HTML document is not named beginning with a verb. Rather, in this course,the name of the main function reflects the name of the js "library" file that holds it.
  • We could also have functions embedded in our HTML pages but that is not the industry standard. JavaScript code should go into a different file that is just JavaScript code.
  • However, if we want to move JavaScript out of HTML we have to use functions.

  • Is that it? Not quite, we have to do one more thing. We have to tell the browser where the file with the JavaScript is located.
  • Use the Firefox "View->Page Source" menu item and view the top of the file we're looking at right now. You should see this line:
  • That line in the <head> tag lets the browser know where to look for the file containing the function. It is also a <script> tag but this one just refers to a file. When the browser sees a JavaScript function call it looks for the function on the page. If it can't find it there then it looks for all the pages referred to by tags like this.
