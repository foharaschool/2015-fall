Lab Steps - Part 2

  1. Open this file in Firefox and jEdit.
  2. Add your name, section, and email.
  3. Locate the main function named lab12firstFunctionCallPart2().
  4. Lab Steps:  (remember to program in versions, testing each version as you go)
    1. Create a new function below the main function lab12firstFunctionCallPart1() but above the main function lab12firstFunctionCallPart2() (and its comment block).
    2. Give the new function a good name that indicates that the function will be outputting a full name. Remember that course standards demand that a function name must begin with a verb, other than for main functions.
    3. Add a comment block above the new function. In this comment block, write a description of what the function does.
    4. Create three variables within the new function.
      1. One for a first name
      2. One for a middle initial
      3. One for a last name.

      Note that these three variables will be used only within this new function, and thus it is best programming practice to set them up within the function. These are officially known as "local" variables--more on this in the next Learning Unit.

    5. The function will then prompt the user for first name, the middle initial, and the last name, one at a time, of course.
    6. Then output the user's full name to the page with a document.write(). Don't add a "<br />" tag to the beginning or the end of the name. Add a period (.) to the end of the middle initial.
    7. Locate the main function named lab12firstFunctionCallPart2().
    8. In the main function lab12firstFunctionCallPart2(), output the first half of a greeting. You could do something like, "Hello, ".
    9. Then call (aka invoke, aka run) the function that you wrote by writing its name. Remember to add the (); at the end of the name of the function. Even if there is nothing to be placed inside the parentheses, they are a part of the name of a function and must be present.
    10. Next, output the end of a greeting. Something like, ". How are you?" and a "<br />" tag.
    11. Now again call the function that you wrote. This time don't output a greeting before you call it.
    12. Then, output a relevant statement, most anything you want (see example output below).
    13. Before being satisfied that the program works perfectly, do a final test of your program. Your output should look something like the following example output.
