Lab Steps

  1. Open this lab HTML file in Firefox and jEdit.
  2. Add your name, section, and email, just as usual.
  3. Open jEdit and create a new empty file , or, probably easier, do this via Windows Explorer.   Save the new file into the /unit4/jsFiles directory (aka folder) giving the new file the name lab01movingJavaScript.js.
  4. Within the source code for this lab HTML file, lab01movingJavaScript.html, in jEdit, scroll down to the JavaScript code.  Note that, at the moment, it is in the place that we're used to it being.
  5. Copy the JavaScript code, and just the JavaScript code, after the line // YOUR CODE STARTS AFTER THIS LINE:, and paste it into the new file you made named lab01movingJavaScript.js.
  6. Add the function header line before the now copied JavaScript code. The function header will look exactly as shown:
    function lab01movingJavaScript() {
  7. Add this after the end:
    }
  8. We've been indenting our code all semester to get used to where it will be when we have it in a function. The function header line and the final } should be all the way over to the left. Our code within the function will then be indented 4 spaces, just like we've been doing.
  9. Now edit the lab HTML file and delete everything between the <script language="JavaScript" type="text/javascript"> line and the </script> tag. That's right, we don't need any of that //<![CDATA[ stuff anymore. And we don't need the JavaScript code either.
  10. Add the function call to the lab HTML file where you just deleted all the other stuff. The function call looks like this:
        lab01movingJavaScript();
    Note that to call (aka invoke, aka run, aka execute, aka "fire") a function, we only need to use the name of the function--no verb is placed in front of it. And also note that while normally by convention the names of functions begin with a verb, in this course these "main" functions do not follow this convention. Instead, their names have been chosen to reflect the name of the js file that houses the function.
  11. Refresh the lab page. Did it work? Did you still get the output?
  12. To make sure it is working properly, go into the function, and change the hard-coded person name values, save, and then again refresh the Web page to see if the function now produces the revised output.
